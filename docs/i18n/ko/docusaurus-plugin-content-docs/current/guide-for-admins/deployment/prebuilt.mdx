# 사전 빌드 릴리스

사전 빌드된 modernGraphTool 릴리스를 내려받아 `dist/` 폴더 전체를 자신의 서버에 호스팅하는 방식입니다. 수동 업데이트를 감수하는 대신, 언제 어떤 버전이 실행될지 완전히 제어할 수 있습니다.

## 개요

사전 빌드 릴리스는 자체 포함된 정적 사이트입니다. 압축을 풀고, 설정과 데이터를 넣은 뒤, 폴더 전체를 웹 서버에 업로드하면 됩니다. jsDelivr에 대한 런타임 의존성이 없고, 버전 감지 로직도 없으며, 자동 업데이트도 없습니다 — 사용자가 무엇을 보게 될지 정확히 제어할 수 있습니다.

## 언제 선택해야 하나

**적합한 경우:**
- 통제된 업데이트 주기 (새 버전이 언제 제공될지 직접 결정)
- 에어갭, 인트라넷, 또는 기타 jsDelivr에 접근할 수 없는 환경

**적합하지 않은 경우:**
- 릴리스마다 파일을 다시 업로드하고 싶지 않은 운영자

## 설치

1.  **다운로드** [최신 릴리스 zip](https://github.com/potatosalad775/modernGraphTool/releases)을 내려받습니다.
2.  **zip 압축 해제.** 압축 해제된 폴더는 완전한 사이트이며, 배포에 필요한 모든 것이 들어 있습니다.
3.  **설정 파일 커스터마이징:**
    - `config.js` — 애플리케이션 설정 ([페이지 커스터마이징](../customize-page) 참고)
    - `theme.css` — 그래프 색상 테마 ([페이지 스타일 커스터마이징](../customize-page#customizing-page-styles-themecss) 참고)
    - `data/` — 측정 데이터 및 타겟 커브 ([데이터 관리](../manage-data) 참고)
4.  **폴더 전체를 웹 서버에 업로드**합니다 (FTP 또는 기타 파일 전송 방식). 끝입니다 — 브라우저에서 페이지를 열어 로드되는지 확인하세요.

## 폴더 구조

릴리스를 압축 해제하면 다음과 같은 구조가 보입니다:

```
dist/                     # (또는 압축 해제한 릴리스 폴더)
├── config.js             # 애플리케이션 설정 — 사용자 편집
├── theme.css             # 그래프 색상 테마 — 사용자 편집
├── index.html            # HTML 진입점 — 메타데이터 편집 가능
├── .htaccess             # Apache SPA 폴백
├── data/
│   ├── phones/           # 측정 데이터 파일 (.txt) — 사용자 편집
│   ├── target/           # 타겟 커브 데이터 파일 (.txt) — 사용자 편집
│   ├── phone_book.json   # 기기 목록 정의 — 사용자 편집
│   ├── Bounds U.txt      # Preference Bound 상한 (선택) — 사용자 편집
│   └── Bounds D.txt      # Preference Bound 하한 (선택) — 사용자 편집
├── assets/               # 이미지, 문자열 등 — 사용자 편집
└── _app/                 # 빌드된 애플리케이션 번들 — 편집 금지
```

**사용자 편집** 표시가 있는 파일은 안전하게 수정할 수 있습니다. `_app/` 폴더에는 빌드된 애플리케이션이 들어 있으며 그대로 두어야 합니다 — 릴리스를 업데이트할 때마다 덮어쓰여집니다.

## 앱 업데이트

사전 빌드 릴리스는 수동 업데이트가 필요합니다. 안전한 절차:

1.  **현재 배포를 백업하세요.** `config.js`, `theme.css`, `data/`, `assets/`, 그리고 커스터마이즈된 `index.html`의 사본을 보관합니다.
2.  **새 릴리스 다운로드** — [릴리스 페이지](https://github.com/potatosalad775/modernGraphTool/releases)에서 내려받아 압축을 해제합니다.
3.  **커스터마이즈된 파일을 다시 복사해 넣기:**
    - `config.js`
    - `theme.css`
    - `data/` (폴더 전체 — phones, targets, `phone_book.json`, bounds)
    - `assets/` (폴더 전체)
    - `index.html` — **커스터마이즈한 경우에만** (OG 태그, 분석 등). 그렇지 않다면 업스트림 개선 사항을 받기 위해 새 릴리스의 `index.html`을 그대로 사용하세요.
4.  **병합된 폴더를 서버에 업로드**하여 기존 배포를 덮어쓰세요.
5.  **[변경 이력](../../changelog)**에서 주의가 필요한 config 스키마 변경이 있는지 확인하세요.

:::warning[데이터를 덮어쓰지 마세요]
가장 흔한 업데이트 실수는 새 릴리스를 라이브 배포 위에 그대로 압축 해제하는 것입니다. 이렇게 하면 `config.js`, `data/` 등이 기본 릴리스 템플릿으로 덮어쓰여집니다. 항상 먼저 백업하거나 복사해 두세요.
:::

## 문제 해결

### 기기 선택기에 데이터가 표시되지 않음

**원인:** `config.js`의 `PATH` 설정이 실제 폴더 레이아웃과 맞지 않거나, `phone_book.json`이 없거나 잘못되었음.

**해결:** `PATH`의 경로 (`PHONE_MEASUREMENT`, `TARGET_MEASUREMENT`, `PHONE_BOOK`)가 실제 파일을 가리키는지 확인하세요. 브라우저의 네트워크 탭을 열어 어떤 요청이 404인지 확인합니다. 예상 레이아웃은 [데이터 관리](../manage-data)를 참고하세요.

### 로컬에서는 동작하는데 서버에서는 안 됨

**원인:** 대소문자 구분. Windows와 macOS 파일 시스템은 기본적으로 대소문자를 구분하지 않지만, 대부분의 Linux 웹 서버는 구분합니다. 디스크 상의 `Harman IE 2019v2.txt` 파일은 `harman ie 2019v2.txt` 요청과 일치하지 않습니다.

**해결:** 파일명을 `phone_book.json` 항목과 정확히 일치시키세요 (대소문자 포함).

### Preference Bound가 렌더링되지 않음

**원인:** `Bounds U.txt`와 `Bounds D.txt`가 `data/`에 없음.

**해결:** 두 파일을 `data/`에 직접 배치하세요 (하위 폴더가 아님). 파일 형식은 [Preference Bound 기능 문서](../../features/preference-bound)를 참고하세요.
