기기 로드하기

기기(Device) 패널은 그래프에 어떤 데이터를 띄울지 고르는 핵심 영역입니다. 하단 패널 전환 메뉴에서 첫 번째 아이콘을 누르거나, 데스크톱이라면 숫자 1 키를 눌러 기기 패널을 열 수 있습니다.

헤더와 두 개의 목록 창​

기기 패널은 맨 윗줄에 스크롤과 무관하게 고정되는 **헤더 바(Header bar)**가 있고, 그 아래로 최대 두 개의 세로 열(Pane)이 나란히 자리합니다.

  • 브랜드 창 (Brand pane) (왼쪽) — 현재 데이터베이스에 등록된 모든 제조사(브랜드) 이름이 수직 리스트로 나열됩니다.
  • 기기 창 (Phone pane) (오른쪽) — 실제 헤드폰 모델 이름이 나열된 스크롤 목록입니다. 왼쪽 창에서 어떤 브랜드를 선택했는지, 그리고 상단 검색창에 어떤 글자를 입력했는지에 따라 실시간으로 필터링됩니다.

검색 필드는 헤더 정중앙에 있으며 두 창 모두에 영향을 줍니다. 왼쪽 브랜드 창을 보고 있든 오른쪽 기기 창을 보고 있든, 검색창에 입력하면 즉시 전체 목록에서 결과를 찾아냅니다.

화면 폭이 넓은 기기에서는 두 창이 좌우로 나란히 보입니다. 반면 좁은 모바일 화면(또는 데스크톱에서 패널 영역을 좁게 줄여둔 경우)에서는 공간이 부족해 한 번에 하나의 창만 표시됩니다. 이때는 헤더 검색창 옆에 나타나는 브랜드(Brands) / 기기(Devices) 탭 버튼으로 두 창을 오가면 됩니다.

브랜드 필터링​

왼쪽 창에서 특정 브랜드 이름을 클릭하면, 오른쪽 창 목록이 즉시 해당 브랜드의 모델만 남기고 정리됩니다. 다른 브랜드를 추가로 클릭하면 그 브랜드의 기기도 목록에 합류합니다. 즉, 브랜드 필터는 교집합이 아닌 합집합(Additive) 방식으로 작동합니다. Sennheiser와 Sony를 동시에 누르면 두 회사 제품이 모두 오른쪽 창에 나타납니다. 선택된 브랜드는 항목 왼쪽에 두꺼운 테두리 선이 생기고 글자색이 진해져 시각적으로 구분됩니다.

브랜드를 하나라도 선택한 상태라면, 패널 최상단에 선택된 브랜드 지우기(Clear selected brands) 버튼이 나타납니다. 이 버튼을 누르면 적용된 모든 브랜드 필터가 풀리고 전체 기기 목록이 다시 나타납니다.

팁

브랜드를 하나도 고르지 않은 기본 상태에서는, 모든 브랜드의 기기가 우측 창에 함께 표시됩니다. 대다수 사용자는 1) 먼저 브랜드를 몇 개 골라 1차로 범위를 좁힌 뒤 스크롤로 탐색하거나, 2) 브랜드 탭은 무시하고 처음부터 검색창에 기기 이름을 입력합니다. 두 방식 모두 잘 작동합니다.

검색 기능​

헤더 바의 검색창은 기기 이름의 일부만 입력해도 실시간으로 부분 문자열(Substring)을 매칭해 결과를 좁힙니다. 이 검색창은 브랜드 필터와 함께 작동합니다. 왼쪽 창에서 두 개의 브랜드를 미리 골라두고 검색창에 글자를 입력하면, 그 두 브랜드 안에서만 검색어를 찾습니다.

검색어 입력란의 'X' 버튼을 누르거나 백스페이스로 글자를 모두 지우면 원래 전체 목록으로 돌아옵니다.

기기 추가 및 삭제​

목록에서 특정 기기 이름을 클릭하면 해당 기기 데이터가 그래프에 추가됩니다. 중앙 캔버스에 색이 다른 꺾은선 커브가 나타나고, 패널 목록의 기기 항목은 활성화 표시로 왼쪽 테두리가 두꺼워지며 제품 리뷰 점수, 가격, 리뷰 본문 링크, 구매처 링크 같은 추가 메타데이터가 아래로 펼쳐집니다. 로드된 기기를 한 번 더 클릭하면 화면에서 해당 모델이 빠지는 삭제가 이뤄집니다 (단, 사이트 운영자가 특정 기준 커브를 강제로 고정해 둔 상태라면 그 커브는 빠지지 않을 수 있습니다).

한 번에 화면에 띄울 수 있는 기기 개수에는 시스템적 제한이 없습니다. 다만 선이 너무 많아지면 데이터를 제대로 읽기 어렵습니다. 보통 서너 개 정도의 커브를 한 화면에 띄우는 것이 가장 적절합니다.

기기 이름을 클릭했을 때 옆에 작은 로딩 스피너(Spinner) 아이콘이 잠깐 나타날 수 있습니다. 서버에서 텍스트 파일을 가져오고 있다는 뜻이며, 개별 FR 데이터 파일 용량이 매우 작기 때문에 보통 거의 즉시 그래프가 표시됩니다.

교차 사이트 통합 검색 (Cross-site search)​

접속한 사이트가 squig.link에 속하고 있다면, CrinGraph에 기반하고 있는 대다수의 다른 리뷰어 사이트와 마찬가지로, 기기 목록 창 맨 아래에 다른 리뷰어 사이트에서 가져온 검색 결과가 추가로 표시됩니다. 이 결과는 검색창에 입력한 글자와 일치하는 결과가 있을 때만 나타나며, 찾는 헤드폰 모델 데이터를 현재 사이트 운영자는 측정하지 않았지만 다른 리뷰어가 측정해 둔 경우 해당 사이트로 클릭 한 번에 이동할 수 있도록 연결해 줍니다. 자세한 내용은 squig.link 연동 문서를 참고하세요.

squig.link 네트워크에 소속되지 않은 독립 사이트 환경에서는 이 교차 사이트 결과 영역 자체가 나타나지 않습니다.

내 컴퓨터의 자체 측정 파일 직접 업로드하기​

기기 패널에서 빠져나와 그래프(Graph) 패널로 넘어간 뒤, 선택된 목록(Selection list)의 스크롤을 가장 아래까지 내려보세요. 그곳에 두 개의 버튼으로 구성된 Graph Uploader가 있습니다.

  • FR 업로드 (Upload FR) — 내 컴퓨터의 파일을 새로운 '헤드폰 기기 측정값' 그래프로 띄웁니다.
  • 타겟 업로드 (Upload target) — 내 컴퓨터의 파일을 단순 측정값이 아닌 '가이드라인 타겟 커브' 용도로 그래프에 띄웁니다.

띄우려는 데이터 성격에 맞는 버튼을 고르고 컴퓨터에서 .txt 측정 파일을 선택하면, 사이트 내장 데이터베이스에서 기기를 고른 것과 동일한 방식으로 선이 화면에 겹쳐 나타납니다. 사용 가능한 파일 형식은 레거시 CrinGraph 시절부터 널리 쓰이던 표준 2열 구조의 텍스트 파일(주파수\t음압레벨(SPL))입니다. modernGraphTool 역시 설치형 데이터베이스 서버 내부적으로 같은 포맷 파일을 읽어 그래프를 그립니다. 직접 업로드 기능으로 다음과 같은 작업을 할 수 있습니다.

  • 검색으로 찾은 다른 사람의 측정 데이터(또는 직접 측정한 데이터)를 현재 사이트 내부 데이터와 겹쳐 비교합니다.
  • 오디오 커뮤니티에서 구한 사용자 제작 타겟 커브 파일을 올려두고 사이트 기본 탑재 타겟 곡선들과 나란히 비교합니다.
  • (리뷰어용) 측정 txt 파일을 웹서버나 깃허브에 업로드하기 전, 화면에 띄워 이상이 없는지 최종 검수를 거칩니다.

업로드된 파일 데이터는 현재 열린 탭이 닫히기 전까지만 브라우저 메모리에 남습니다. — 외부 서버로 전송되지 않습니다. 탭을 닫거나 새로고침(F5)을 누르면 업로드한 선은 모두 사라집니다.

그래프 영역 위에 헤드폰 커브 선을 하나라도 띄우셨나요? 선이 하나라도 올라간 시점부터는 각 선을 정교하게 조작하는 커브 전용 컨트롤 버튼이 본격적으로 활용됩니다. 커브 다루기 문서에서 자세한 조작법을 확인하세요.
