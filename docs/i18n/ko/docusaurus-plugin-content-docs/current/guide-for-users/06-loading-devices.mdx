# 기기 로드하기

**기기(Device) 패널**은 여러분이 그래프 위에 무엇을 띄워 그릴지 선택하는 핵심 구역입니다. 하단의 패널 전환 메뉴에서 첫 번째 아이콘을 누르거나, 데스크톱 환경이라면 숫자 **1** 키를 눌러 기기 패널을 열 수 있습니다.

## 헤더와 두 개의 목록 창

기기 패널은 맨 윗줄에 스크롤을 내려도 따라다니는 고정 **헤더 바(Header bar)**를 가지고 있으며, 그 아래로 최대 두 개의 세로 열(Pane)이 나란히 자리합니다:

- **브랜드 창 (Brand pane)** (왼쪽) — 현재 데이터베이스에 등록된 모든 제조사(브랜드) 이름이 수직 리스트로 나열됩니다.
- **기기 창 (Phone pane)** (오른쪽) — 실제 헤드폰 모델들의 이름이 나열된 스크롤 목록입니다. 왼쪽 창에서 어떤 브랜드를 선택했는지, 그리고 상단 검색창에 어떤 글자를 쳤는지에 따라 실시간으로 필터링되어 내용물이 줄어듭니다.

검색 필드는 헤더 정중앙에 위치하며 두 창 모두에 영향을 미칩니다. 여러분이 왼쪽 브랜드 창을 보고 있든 오른쪽 기기 창을 보고 있든 상관없이, 검색창에 타이핑하면 즉시 전체 목록을 뒤져 결과를 찾아냅니다.

화면 폭이 넓은 기기에서는 이 두 개의 창이 좌우로 나란히 보입니다. 반면 좁은 모바일 화면(또는 데스크톱에서 패널 영역을 아주 좁게 줄여 놓은 경우)에서는 공간이 부족하여 한 번에 하나의 창만 표시됩니다. 이럴 때는 헤더 검색창 바로 옆에 나타나는 **브랜드(Brands) / 기기(Devices)** 탭 버튼을 눌러 두 창 사이를 휙휙 오가면 됩니다.

## 브랜드 필터링

왼쪽 창에서 특정 브랜드 이름을 하나 클릭해 보세요. 오른쪽 창의 목록이 즉시 해당 브랜드가 만든 모델들만 남기고 싹 정리됩니다. 다른 브랜드를 추가로 클릭하면 그 브랜드의 기기들도 목록에 합류합니다. 즉, 브랜드 필터는 교집합이 아닌 **합집합(Additive)** 방식으로 작동합니다. _Sennheiser_와 _Sony_를 동시에 누르면 두 회사의 제품이 모두 오른쪽 창에 나타납니다. 여러분이 선택한 브랜드들은 항목 왼쪽에 두꺼운 테두리 선이 생기고 글자색이 진해져 시각적으로 뚜렷하게 구분됩니다.

브랜드를 하나라도 선택한 상태라면, 패널 최상단에 **선택된 브랜드 지우기(Clear selected brands)** 버튼이 나타납니다. 이 버튼을 누르면 걸려 있던 모든 브랜드 필터 자물쇠가 풀리고 전체 기기 목록이 다시 나타납니다.

:::tip
브랜드를 단 하나도 고르지 않은 기본 상태에서는, 세상 모든 브랜드의 기기가 우측 창에 전부 뒤섞여 표시됩니다. 대다수의 유저들은 1) 먼저 브랜드를 몇 개 찍어서 1차로 범위를 좁힌 뒤 스크롤을 내리거나, 2) 브랜드 탭은 아예 무시하고 처음부터 검색창에 기기 이름을 친 뒤 바로 엔터를 누릅니다. 두 방식 모두 완벽하게 작동합니다.
:::

## 검색 기능

헤더 바의 검색창은 기기 이름의 일부만 쳐도 실시간으로 부분 문자열(Substring)을 매칭해 결과를 솎아냅니다. 이 검색창은 브랜드 필터와 훌륭하게 연계되어 작동합니다: 만일 왼쪽 창에서 두 개의 브랜드를 미리 골라두고 검색창에 글자를 친다면, 오직 그 두 브랜드 안에서만 검색어를 뒤집니다.

검색어 입력란의 'X' 버튼을 누르거나 백스페이스로 글자를 모두 지우면 원래의 전체 목록으로 즉시 돌아옵니다.

## 로드(Load)와 언로드(Unload)

목록에서 특정 기기 이름을 클릭하면 해당 기기의 데이터가 그래프로 **로드(Load)**됩니다. 중앙 캔버스에 형형색색의 꺾은선 커브가 나타나고, 패널 목록의 기기 항목은 활성화되었다는 표시로 왼쪽 테두리가 두꺼워지며 제품 리뷰 점수, 권장소비자가격(MSRP), 리뷰 본문 링크, 구매처 숍 링크 같은 추가 메타데이터가 아래로 펼쳐집니다. 로드된 기기를 한 번 더 클릭하면 화면에서 모델이 빠지는 **언로드(Unload)** 처리가 완료됩니다 (단, 사이트 운영자가 특정 기준 커브를 강제로 고정시켜 둔 상태라면 그 커브는 빠지지 않을 수 있습니다).

한 번에 화면에 띄울 수 있는 기기의 개수에는 시스템적 제한이 없습니다. 하지만 선이 너무 많아지면 결국 스파게티 덩어리처럼 변해 데이터를 제대로 읽을 수 없게 됩니다. 보통 **서너 개** 정도의 커브를 한 화면에 띄우는 것이 시각적 교차 비교를 위한 가장 훌륭한 타협점입니다.

기기 이름을 클릭했을 때 옆에 작은 빙글빙글 도는 로딩 스피너(Spinner) 로고가 잠깐 나타날 수 있습니다. 서버에서 텍스트 파일을 가져오고 있다는 뜻이며, 개별 FR 데이터 파일 용량이 워낙 작기 때문에 보통 모래시계를 볼 새도 없이 거의 즉시 그래프가 팝업됩니다.

## 교차 사이트 통합 검색 (Cross-site search)

만약 여러분이 접속한 사이트가 **squig.link** 도메인 호스팅을 사용 중이라면 여타 대다수의 리뷰어 사이트들과 마찬가지로, 기기 목록 창 맨 밑바닥에 내가 지금 접속한 사이트가 아닌 _다른_ 리뷰어 형제 사이트들에서 스크랩해 온 검색 결과가 추가로 뜹니다. 이 외인부대 결과들은 여러분이 검색창에 **타이핑을 하고 일치하는 결과가 있을 때만** 특별히 모습을 드러내며, 내가 찾는 헤드폰 모델 데이터를 이 사이트 운영자는 측정한 적이 없지만 옆 동네 리뷰어는 측정해 둔 경우 그쪽 사이트로 클릭 한 번에 공간 이동을 할 수 있게 해주는 마법의 문입니다. 자세한 전말은 [squig.link 연동](../features/squiglink-integration) 문서를 참고하세요.

물론 squig.link 네트워크 연합에 소속되지 않은 독립 사이트 환경에서는 이 교차 사이트 결과 영역 자체가 아예 나타나지 않습니다.

## 내 컴퓨터의 자체 측정 파일 직접 업로드하기 {#uploading-your-own-measurement}

방금까지 보던 기기 패널에서 빠져나와 **그래프(Graph) 패널**로 넘어간 뒤, 선택된 목록들(Selection list)의 스크롤을 맨 밑바닥까지 끝까지 내려보세요. 그곳에 두 개의 직관적인 버튼을 가진 **Graph Uploader**가 숨어 있습니다:

- **FR 업로드 (Upload FR)** — 내 컴퓨터의 파일을 새로운 '헤드폰 기기 측정값' 그래프에 띄웁니다.
- **타겟 업로드 (Upload target)** — 내 컴퓨터의 파일을 단순 측정값이 아닌 '가이드라인 타겟 커브'의 용도로 그래프에 띄웁니다.

여러분이 띄우고자 하는 데이터의 성격에 맞는 버튼을 하나 고르고 컴퓨터 안에서 `.txt` 측정 파일을 고르면, 사이트 내장 데이터베이스에서 기기를 하나 고른 것과 완전히 똑같이 여러분의 선이 화면에 겹쳐 나타납니다. 사용 가능한 파일 형식은 레거시 CrinGraph 시절부터 널리 쓰이던 표준 2열 구조의 텍스트 파일(`주파수\t음압레벨(SPL)`)입니다. modernGraphTool 역시 설치형 데이터베이스 서버 내부적으로는 똑같은 포맷 파일을 읽어들여 캔버스를 그립니다. 여러분은 이 직접 업로드 기능을 이용해 이런 곡예들을 부릴 수 있습니다:

- 구글링으로 찾아낸 다른 사람의 측정 데이터(또는 내가 직접 측정한 데이터)를 현재 사이트 내부 데이터와 포개어 비교해 봅니다.
- 오디오 커뮤니티에서 구한 사제 타겟 커브 파일을 올려두고 사이트 내 기본 탑재 타겟 곡선들과 나란히 켜 보며 비교합니다.
- (리뷰어용) 내 측정 txt 파일을 웹서버나 깃허브에 밀어 올리기 전, 화면에 던져 넣고 찌그러진 곳은 없는지 최종 산출물 검수를 거칩니다.

업로드된 파일 데이터는 오직 여러분의 현재 열린 탭을 끄기 전까지만 브라우저 메모리에 고여있습니다. — **외부 서버 어딘가로 남몰래 전송되는 일은 없습니다.** 탭을 닫거나 새로고침(F5)을 누르면 업로드했던 선들은 전부 증발합니다.
