# 인터페이스 둘러보기

원리와 개념 설명은 여기까지입니다. 이 문서부터 가이드라인은 화면에 나타나는 버튼과 조작법을 다룹니다. 본격적으로 세부 조작법으로 들어가기 전에, 도구의 전체적인 큰 그림을 먼저 그려보겠습니다.

## 화면의 세 가지 영역

사용 중인 기기 사이즈와 무관하게, 도구 화면은 항상 세 개의 큰 영역으로 나뉩니다:

1. **상단 바 (Top bar)** — 화면 맨 위를 지나는 얇은 막대입니다. 사이트의 제목과 운영자가 설정해 둔 외부 링크들이 표시됩니다. 만일 사이트가 squig.link 소속이라면, 다른 리뷰어들의 사이트로 바로 넘어갈 수 있는 선택 메뉴가 여기에 나타납니다.
2. **그래프 영역 (Graph area)** — 주파수 응답 커브들이 실제로 그려지는 거대한 캔버스입니다. 여러분의 시선이 가장 오래 머무는 공간이죠.
3. **패널 영역 (Panel area)** — 헤드폰 기기 데이터를 로드하고, 그래프를 설정하거나 EQ를 적용할 수 있는 모든 컨트롤 버튼들이 모인 곳입니다. 여러분이 "능동적으로 조작하는" 모든 작업은 이 패널 영역 안에서 일어납니다.

창 크기(화면 폭)에 따라 그래프 영역과 패널 영역이 화면을 나눠 갖는 방식(레이아웃)은 변하지만, 이 세 가지 영역이 존재한다는 기본 틀 자체는 변하지 않습니다.

## 네 개의 패널

패널 영역은 길게 늘어진 하나의 화면이 아닙니다. 작은 **패널 전환 메뉴(Menu carousel)**(네 개의 아이콘 묶음)를 눌러 서로 전환할 수 있는 네 개의 독립된 패널로 나뉘어져 있습니다. 각각의 패널은 부여받은 목적이 뚜렷합니다:

| 패널 명칭 | 주요 용도 | 상세 가이드 |
|---|---|---|
| **기기 (Device)** | 데이터베이스를 탐색 및 검색하고, 헤드폰 커브를 화면에 불러옵니다. | [기기 로드하기](./06-loading-devices.mdx) |
| **그래프 (Graph)** | 타겟을 선택하고, 그래프 환경을 설정하며, 이미 화면에 불려진 커브 선형들을 관리합니다. | [그래프 컨트롤](./08-graph-controls.mdx) & [커브 다루기](./07-working-with-curves.mdx) |
| **이퀄라이저 (Equalizer)** | 나만의 EQ 프로파일을 만들고, 소리를 미리 들어보거나 하드웨어 장비로 EQ를 직접 전송합니다. | [EQ 적용](./10-equalizing.mdx) |
| **기타 (Misc)** | 화면 테마, UI 번역 언어, 사이트 소개 및 앱 시스템 정보를 관리합니다. | [외관과 언어](./12-appearance-and-language.mdx) |

화면에는 한 번에 하나의 패널만 열려 있습니다. 패널 간 전환은 즉각적입니다. 그래프는 그 자리에 가만히 고정된 채, 선택한 패널 내용물만 옆에서 미끄러지듯 들어옵니다.

:::tip[패널 사이를 단축키로 이동하세요]
데스크톱 환경이라면 키보드 숫자 **1**, **2**, **3**, **4** 키를 눌러 기기, 그래프, 이퀄라이저, 기타 패널로 즉시 이동할 수 있습니다. 단축키 조합 없이 그냥 숫자만 누르면 됩니다.
:::

## 데스크톱 레이아웃

브라우저 창의 너비가 1000 픽셀 이상으로 넉넉할 때, modernGraphTool은 **데스크톱 레이아웃**으로 작동합니다:

- 그래프 영역은 화면의 **왼쪽**에 배치되며 가장 넗은 면적을 차지합니다.
- 패널 영역은 화면의 **오른쪽**에 위치합니다. 이 영역은 기본 너비가 정해져 있지만, 마우스로 크기를 자유롭게 조절할 수 있습니다.
- 두 영역 사이에는 얇은 **드래그 구분선(Drag divider)**이 존재합니다. 이 선을 클릭한 채 좌우로 끌면 그래프와 패널이 차지하는 공간 비율을 조절할 수 있습니다.
- 그래프 바로 밑에는 (정규화, 스무딩, 스크린샷 등을 조작하는) **그래프 툴바**가 놓이며, 그 아래로 주파수 튜토리얼 띠와 아주 조그마한 **키보드 단축키 안내 바**가 그래프 칼럼 맨 밑바닥 피날레를 장식합니다.
- 조심해야 할 점: 화면에 로드된 커브 객체들을 관리하는 **선택 목록(Selection list)**은 그래프 바로 밑에 있지 않습니다. 오른쪽 구역의 **그래프(Graph) 패널** 안에 들어있습니다. 따라서 특정 커브 선의 옵션을 조작하고 싶다면 가장 먼저 "그래프 패널" 탭표시로 넘어가야 합니다.

## 모바일 레이아웃

브라우저 창 너비가 1000 픽셀보다 좁아지면 (스마트폰, 세로로 든 태블릿, 작게 줄여 둔 PC 창 등), modernGraphTool은 **모바일 레이아웃**으로 레이아웃을 완전히 교체합니다:

- 그래프 영역은 화면의 **가장 위쪽(상단)**에 고정됩니다.
- 패널 영역은 그래프 밑으로 내려와 나머지 화면 전체를 차지합니다. 화면 스와이프와 터치 조작은 모두 여기서 진행됩니다.
- 패널을 이리저리 오갈 수 있는 전환 메뉴는 패널 영역 **맨 밑바닥(하단)**으로 내려가 엄지손가락이 닿기 쉬운 곳에 자리 잡습니다.
- (정규화, 스무딩 등을 조작하던) **그래프 툴바**는 좁은 화면을 아끼기 위해 접이식 아코디언 메뉴 안에 숨겨진 채 그래프 패널 최상단으로 옮겨집니다. 탭을 눌러 펼쳐야 내용물이 보입니다.
- 모바일 레이아웃에는 드래그 구분선이 없습니다. 그래프와 패널 영역의 화면 점유 비율은 단단히 고정되어 있습니다.

선택 목록(Selection list)은 데스크톱이든 모바일이든 상관없이 항상 굳건히 "그래프 패널" 안쪽에 들어 있습니다. 따라서 "내 커브 선들을 조작하려면 우선 그래프 패널로 들어가야 한다"는 멘탈 모델은 모든 환경에서 완벽히 동일합니다.

모바일 패널 구역에서는 하단의 메뉴 전환 아이콘을 직접 탭해서 패널을 바꿀 수도 있고, 패널 영역 안에서 손가락을 **좌우로 쓸어 넘겨(스와이프)** 패널 간을 유연하게 오갈 수도 있습니다.

## 키보드와 포커스 조작

UI 내부의 대다수 버튼 기능은 마우스 없이 키보드만으로도 접근 가능합니다:

- **Tab** 키를 누르면 다음 상호작용 가능한 요소로 포커스가 넘어가고, **Shift+Tab**을 누르면 이전 요소로 돌아옵니다.
- **Enter** 나 **Space** 키는 현재 포커스된 컨트롤(버튼)을 실행합니다.
- **Ctrl+Z** 및 **Ctrl+Shift+Z** (또는 **Ctrl+Y**) 구조는 커브를 로드하거나 지우거나 수정한 행동 내역들을 실행 취소 / 다시 실행 처리합니다.
- 데스크톱 환경일 때 숫자 **1 / 2 / 3 / 4** 키를 눌러 4개의 패널을 즉시 넘나들 수 있습니다.

키보드 조작성 및 시각장애인용 스크린 리더(Screen-reader) 지원은 완성을 위해 계속 다듬어지고 있는 영역입니다. 탭으로 움직여도 포커스가 닿지 않는 컨트롤 버튼이나 스피커로 읽히지 않는 레이블을 마주하셨다면 소중한 버그 리포트를 기다리겠습니다.
