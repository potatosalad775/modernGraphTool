# 환경 구성하기

이 가이드에서는 modernGraphTool을 사용하고 관리하기 위한 기본적인 환경 설정 방법을 안내합니다.

## 준비물

1.  **modernGraphTool 프로젝트 파일**: 주파수 응답 데이터를 시각화하고 분석하기 위한 핵심 기능이 포함된 빌드된 애플리케이션 파일입니다.
2.  **텍스트 에디터**: 각종 설정 파일(`.js`, `.css`, `.json`)을 수정하기 위한 프로그램입니다. [Visual Studio Code (VS Code)](https://code.visualstudio.com/) 사용을 권장합니다.

## 옵션 A: 사전 빌드된 릴리스 사용 (권장)

가장 간단한 방법입니다. 프로그래밍 지식이 필요하지 않습니다.

1.  [modernGraphTool GitHub 릴리스 페이지](https://github.com/potatosalad775/modernGraphTool/releases)에서 [최신 릴리스 zip 파일](https://github.com/potatosalad775/modernGraphTool/releases)을 다운로드합니다.
2.  다운로드한 zip 파일의 압축을 해제합니다. 압축 해제된 폴더에는 배포에 필요한 모든 파일이 포함되어 있습니다.
3.  필요에 맞게 설정 파일을 수정합니다:
    - `config.js` — 애플리케이션 설정 ([페이지 커스터마이징](./customize-page) 참조)
    - `theme.css` — 그래프 색상 테마 ([페이지 스타일 개인화하기](./customize-page#페이지-스타일-개인화하기-themecss) 참조)
    - `data/` — 측정 데이터 및 타겟 커브 ([데이터 관리](./manage-data) 참조)
4.  FTP 또는 기타 파일 전송 방법을 사용하여 전체 폴더를 웹 서버에 업로드합니다.

## 옵션 B: 소스에서 직접 빌드 (개발자용)

소스 코드를 수정하거나 최신 개발 버전에서 빌드하고 싶은 경우:

1.  시스템에 [Node.js](https://nodejs.org/) (v18 이상)가 설치되어 있는지 확인합니다.
2.  저장소를 클론합니다:
    ```bash
    git clone https://github.com/potatosalad775/modernGraphTool.git
    cd modernGraphTool
    ```
3.  의존성을 설치합니다:
    ```bash
    npm install
    ```
4.  설정을 수정합니다:
    - `static/config.js` — 애플리케이션 설정
    - `static/theme.css` — 그래프 색상 테마
    - `static/data/` — 측정 데이터 및 타겟 커브
5.  프로젝트를 빌드합니다:
    ```bash
    npm run build
    ```
6.  빌드된 결과물은 `dist/` 폴더에 생성됩니다. 이 폴더를 웹 서버에 배포하면 됩니다.

:::tip[로컬 미리보기]
배포 전에 변경 사항을 로컬에서 미리 확인할 수 있습니다:
- **소스에서 실행**: `npm run dev`를 실행하면 `http://localhost:5173`에서 개발 서버가 시작됩니다.
- **빌드 결과물 확인**: `npm run preview`를 실행하면 `dist/` 폴더를 로컬에서 서빙합니다.
:::

## 텍스트 에디터 설치

다양한 텍스트 에디터가 있지만, 코드 하이라이팅, 자동 완성 등 편리한 기능을 제공하는 [Visual Studio Code (VS Code)](https://code.visualstudio.com/) 사용을 권장합니다.

VS Code는 누구나 무료로 다운로드하여 사용할 수 있으며, Windows, macOS, Linux를 모두 지원합니다.

1.  [VS Code 웹사이트](https://code.visualstudio.com/)에 접속하여 사용 중인 운영체제에 맞는 설치 파일을 다운로드하고 설치합니다.
2.  VS Code에서 modernGraphTool 프로젝트 폴더(또는 압축 해제한 릴리스 폴더)를 엽니다.
3.  이제 설정 파일을 수정하고 데이터를 관리할 수 있습니다.

## 프로젝트 구조

릴리스 압축 해제 후(또는 소스에서 빌드한 후)의 주요 파일 및 폴더 구조는 다음과 같습니다:

```
dist/                     # (또는 압축 해제한 릴리스 폴더)
├── config.js             # 애플리케이션 설정
├── theme.css             # 그래프 색상 테마
├── index.html            # HTML 진입점 (메타데이터 수정 가능)
├── data/
│   ├── phones/           # 측정 데이터 파일 (.txt)
│   ├── target/           # 타겟 커브 데이터 파일 (.txt)
│   └── phone_book.json   # 기기 목록 정의
└── assets/               # 정적 자산 (이미지, 문자열 등)
```

:::info[소스 빌드 사용자 참고]
소스에서 빌드하는 경우, 수정 가능한 파일은 `static/` 폴더에 있습니다. 소스의 HTML 진입점은 `src/app.html`(SvelteKit 템플릿)입니다. 빌드 후 이 파일들은 컴파일되어 `dist/`에 출력됩니다.
:::
