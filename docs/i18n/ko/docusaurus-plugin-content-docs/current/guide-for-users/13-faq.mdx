# FAQ

자주 묻는 질문(FAQ)에 대한 간략한 답변입니다. 원하는 내용이 없다면 기능별 심층 설명이 있는 [기능](../category/features) 섹션이나 메인 [개요](../intro) 페이지를 확인하십시오.

## 측정 및 해석

### 왜 같은 헤드폰인데 두 사이트의 측정치가 다르게 나오나요?

서로 다른 측정 장비나 보정 곡선을 사용하기 때문입니다. 측정 장비의 물리적 특성과 보정 곡선의 수학적 계산이 모두 그래프 형태를 결정합니다. 어느 한쪽 사이트가 "틀린" 경우는 드뭅니다. 자세한 내용은 [측정 원리](./03-how-measurements-work.mdx)를 읽어보십시오. 실용적인 원칙은 다음과 같습니다. 단일 사이트 내부의 비교는 신뢰할 수 있지만 사이트 간 비교에는 주의가 필요합니다.

### 왜 측정치가 그렇게 울퉁불퉁해 보이나요?

실제 음향 환경은 매우 복잡하므로 헤드폰의 응답 특성에도 굴곡이 생깁니다. 커플러 공진, 헤드폰 자체 드라이버, 이어패드의 밀폐 방식, 그리고 보정 곡선 모두가 파형의 변화를 유발합니다. 이는 불량 장비에서 발생하는 노이즈가 아니며, 시뮬레이션된 고막에 도달하는 소리를 정직하게 보여주는 결과입니다. 어느 대역의 굴곡이 중요한지에 대해서는 [02 · 그래프 읽기](./02-reading-the-graph.mdx) 페이지를 참조하십시오.

### 곡선 주변의 음영 처리된 띠는 무엇인가요?

이는 **샘플 간 편차**를 나타냅니다. 동일한 기기를 여러 번 측정하거나 여러 제품을 측정했을 때 나타나는 편차 범위입니다. 띠가 좁으면 헤드폰의 측정 결과가 일관적임을 의미하고, 띠가 넓으면 착용 방식, 밀폐 정도, 혹은 제품 간 편차에 크게 의존함을 뜻합니다. 샘플 선택기를 통해 곡선별 음영 스타일을 변경할 수 있습니다([곡선 다루기](./07-working-with-curves.mdx) 참조).

### 그래프 전체를 가로지르는 큰 음영 영역은 무엇인가요?

이는 **선호도 경계**입니다. "대부분의 청취자가 만족하는" 범위를 연구를 통해 도출한 것입니다. 곡선이 이 영역 안에 있으면 선호 범위에 속하며, 범위를 벗어난 곡선은 평균적인 선호도보다 더 밝거나 어둡게 들린다는 의미입니다. 그래프 툴바에서 이 영역을 켜거나 끌 수 있습니다. 자세한 내용은 [타겟 및 선호도](./09-targets-and-preferences.mdx)와 전체 기능 문서인 [선호도 경계](../features/preference-bound)에서 확인하십시오.

## 사용자 인터페이스

### 그래프가 이상하게 축소되거나 확대되어 보입니다.

Y축 스케일 설정 때문입니다. 그래프 툴바에서 **Y 스케일 버튼**을 클릭하여 데시벨 범위를 변경할 수 있습니다. 자세한 내용은 [그래프 조작](./08-graph-controls.mdx#y-axis-scale)을 참조하십시오.

### 실수로 곡선을 삭제했습니다. 영영 사라진 건가요?

**Ctrl+Z**를 누르십시오. 데이터 불러오기, 제거, 곡선별 편집 작업은 모두 실행 취소 스택에 저장되므로 여러 단계 이전으로 되돌릴 수 있습니다. **Ctrl+Shift+Z**(또는 **Ctrl+Y**)를 누르면 다시 실행됩니다.

### 내 측정 파일을 업로드할 수 있나요?

네, 목록 맨 아래로 스크롤하여 **그래프 업로더**를 사용할 수 있습니다. CrinGraph 형태의 2열 주파수/음압 텍스트 파일을 끌어다 놓으면 내장된 기기 데이터처럼 불러옵니다. 이 파일은 브라우저 세션에만 유지되며 어떤 서버로도 전송되지 않습니다. [기기 불러오기](./06-loading-devices.mdx#uploading-your-own-measurement)를 확인하십시오.

### 특정 주파수에서 두 헤드폰을 어떻게 비교하나요?

**분석 모드**를 켜고 해당 주파수 위에 마우스를 올려 두 데시벨 값을 모두 읽거나, 정규화 방식을 **주파수 모드**로 변경하고 원하는 주파수를 입력하십시오. 해당 지점에서 곡선이 정렬되며, 다른 주파수 대역에서의 수직 거리가 두 기기의 차이를 나타냅니다. 자세한 내용은 [그래프 조작](./08-graph-controls.mdx)에 있습니다.

### 다크 모드 전환 버튼은 어디에 있나요?

**기타 패널**(네 번째 메뉴 아이콘, 데스크톱에서는 숫자 **4** 키) 상단에 있는 해/달 아이콘입니다. [외관 및 언어](./12-appearance-and-language.mdx#light-and-dark-theme)를 참조하십시오.

## 공유

### 공유 URL이 너무 길고 복잡합니다. 줄일 수 있나요?

URL에 전체 그래프 상태가 포함되어 있어 의도적으로 깁니다. 데이터를 저장하는 서버나 계정이 없기 때문입니다. 짧은 링크를 원한다면 공개 URL 단축 서비스를 사용하십시오. 포럼 게시물이나 채팅 메시지 등에 사용할 때 길이가 길어도 도구 작동에는 문제가 없습니다.

### 다른 사람에게 공유 URL을 보냈는데 제가 보는 화면과 다르게 나온다고 합니다.

공유 URL은 공유 버튼을 클릭했을 때 접속해 있던 특정 사이트를 기준으로 생성됩니다. 만약 수신자가 _다른_ 측정 사이트에서 URL을 열면, 그 사이트의 데이터베이스에 없는 기기 이름은 소리 없이 무시됩니다. 발신자와 수신자가 동일한 측정 사이트를 이용 중인지 확인하십시오.

## 이퀄라이저

### 오디오 미리보기의 재생 버튼을 눌렀는데 아무 소리도 나지 않습니다.

대부분의 브라우저는 사용자가 페이지의 어느 곳이든 클릭하기 전까지 오디오 재생을 차단합니다. 도구의 아무 곳이나 먼저 클릭한 후 다시 재생을 누르십시오. 그래도 작동하지 않으면 브라우저 탭이 음소거되어 있지 않은지 확인하십시오.

### 프리앰프는 왜 필요한가요?

특정 주파수를 디지털로 6데시벨 높이면 전체 신호가 디지털 한계점에 6데시벨만큼 가까워져서 클리핑이 발생할 수 있습니다. 도구가 한계점 아래로 피크를 유지하도록 전체 게인을 낮추는 프리앰프 값을 자동으로 계산합니다. 수동으로 관리할 필요가 없습니다. [이퀄라이저](../features/equalizer)를 참조하십시오.

### DAC나 헤드폰 하드웨어에 직접 이퀄라이저를 전송할 수 있나요?

현재 지원되는 기기라면 가능합니다. 이퀄라이저 패널 안의 **기기 PEQ** 섹션을 여십시오. Chrome과 Edge 환경에서 가장 원활하게 지원됩니다. Safari는 기반이 되는 웹 API를 전혀 지원하지 않습니다. 지원 기기 목록 등 자세한 내용은 [기기 PEQ](../features/device-peq)에서 확인하십시오.

## squig.link 통합

### "지금 구매" 혹은 상점 링크 버튼은 무엇인가요?

이 버튼은 **squig.link**에서 호스팅되는 사이트 중, 현재 선택된 기기가 스폰서 상점 데이터베이스에 존재하는 경우에만 나타납니다. 클릭하면 해당 기기의 구매 페이지로 이동합니다. 버튼이 보이지 않는다면 사이트가 squig.link 소속이 아니거나 상점 데이터베이스에 해당 기기가 없는 것입니다. [squig.link 통합](../features/squiglink-integration)을 참조하십시오.

### 다른 squig.link 사이트들도 검색할 수 있나요?

squig.link 연동 사이트에서는 가능합니다. 기기 패널의 검색창에 입력하면 네트워크에 속한 다른 사이트의 일치하는 기기들이 현재 사이트의 결과 아래에 나타납니다. 해당 결과를 클릭하면 그 사이트로 이동합니다. squig.link 네트워크에 속하지 않은 사이트에서는 교차 검색 결과가 나타나지 않습니다.
