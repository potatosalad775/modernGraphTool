# Overview

modernGraphTool is a Single Page Application (SPA) based on modern web technologies, designed for visualizing and analyzing Frequency Response (FR) data.

## Why modernGraphTool?

modernGraphTool was directly inspired by [CrinGraph](https://github.com/mlochbaum/CrinGraph), which appeared in 2019 and has long been used as a standard tool for visualizing frequency response data. Even now, CrinGraph is widely used and loved for visualizing and comparing FR data.

However, as time went by, those who operating databases gradually needed more complex and diverse functions. Custom HRTF targets with their own preference adjustments are a prime example.

Unfortunately, CrinGraph was not designed with extensibility in mind from the initial design stage, leading to fragmentation issues where numerous derivative versions emerged as users added their own features.

This made code maintenance extremely difficult and made it almost impossible for users without programming knowledge to add new features or import features from other versions.

modernGraphTool aims to solve these problems, inheriting the core features of CrinGraph while introducing modern techniques and a modular design to significantly enhance stability and extensibility.

## Core Principles

### Feature-Rich Out of the Box

modernGraphTool comes with a comprehensive set of built-in features that cover everything from basic graph visualization to advanced equalization and hardware device integration — all configurable through a single `config.js` file.

Unlike fragmented CrinGraph derivatives where adding features required modifying core scripts, modernGraphTool's features are self-contained components that can be enabled, disabled, or configured without risk to the core functionality.

### Easy-to-Use Design

Some CrinGraph derivative versions had problems with UI complexity as various features were added.

Numerous horizontal scroll areas with long list of buttons often impaired usability in mobile environments.

modernGraphTool focuses on user interface (UI) and user experience (UX) design to be intuitive and convenient in both desktop and mobile environments, organizing features into clearly defined panels (Device, Graph, Equalizer, Misc) with a responsive layout.

### Compatibility with Existing Programs

modernGraphTool fully supports the folder structure and file format of FR data used in the existing CrinGraph.

Therefore, you can load the data from your existing CrinGraph directly into modernGraphTool, and compatibility with `squig.link` is also ensured through built-in integration.

## Features

### Frequency Response Graph Visualization
- Normalization options — Hz-based or Midrange Average (300Hz~3kHz)
- Smoothing options — 1/48 oct to 1/3 oct
- Multi-sample measurement display (average or individual samples)
- HpTF (Headphone Transfer Function) deviation display
- Custom HRTF Target support with Tilt & Preference Adjustment

### Headphone / Target Selector
- Search headphones by name or brand
- Headphone / target compensation (set as baseline)
- Variant and channel selection (L, R, L+R, Average)
- Compatible with CrinGraph file structure (`phone_book.json` & `target_manifest`)

### Equalizer & Audio
- Built-in parametric EQ panel with multiple filter types (PK, LSQ, HSQ, Tilt)
- AutoEQ — automatic filter generation from target curves
- Real-time audio playback with EQ applied (Web Audio API)
- Test tone generator
- EQ filter import/export

### Device PEQ Bridge
- Push EQ settings directly to compatible hardware devices
- Connection via USB HID, USB Serial, or Network
- Support for FiiO, Qudelix-5K, Moondrop, WiiM, JDS Labs, and more
- Multi-slot preset management

### Graph Tools
- Share with URL (+ Base62 encoding support)
- Save frequency response graph as PNG screenshot
- Preference bound overlay
- Graph color customization
- Frequency range tutorial for beginners
- Inspection mode for detailed measurement values

### User Interface & Customization
- Multi-Language & Dark Mode support
- User configurable elements (links, watermark, description)
- Graph theme customization via `theme.css` (+ [Theme Generator](/theme-generator))

### squig.link Integration
- Cross-site device search across squig.link network
- Shop links for matching devices
- Analytics integration
- Sponsor banner support
