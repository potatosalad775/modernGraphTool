타겟과 선호도

이 페이지는 타겟이 존재하는 이유에 이은 후속 내용입니다. 타겟이 무엇을 의미하는지 알았으니, 이제 타겟을 선택하고 조정하며 선호도 경계를 읽는 방법을 알아봅니다.

타겟 선택하기​

그래프 패널 상단의 타겟 선택기는 사이트 운영자가 포함한 모든 타겟의 목록을 보여줍니다. 정확한 목록은 배포 환경마다 다릅니다. 두 squig.link 사이트가 반드시 같은 타겟을 제공하지는 않으며, 사이트마다 자체 측정 장비와 철학에 맞춰 목록을 구성하기 때문입니다.

  • 타겟을 클릭하면 그래프에 불러옵니다. 선택 목록에 타겟만의 행이 생기고 점선으로 표시됩니다.
  • 다른 타겟을 클릭하여 추가로 불러옵니다. 타겟들은 서로 독립적이라 여러 개를 겹쳐 비교할 수 있습니다.
  • 불러온 타겟을 선택기에서 다시 클릭하면 제거됩니다.

타겟은 항상 선택 목록의 최상단에 정렬되므로, 불러온 기기들 사이에서도 쉽게 찾을 수 있습니다.

타겟 커스터마이징​

일부 사이트는 파일을 수정하지 않고도 타겟을 사용자의 취향에 맞게 약간 _조정_할 수 있도록 허용합니다. 이를 지원하는 타겟은 선택 목록의 해당 행에 작은 슬라이더 아이콘이 표시됩니다. 아이콘을 클릭하면 몇 가지 필터와 프리셋이 담긴 화면이 열립니다.

정확한 필터는 사이트 설정에 따라 다르지만, 전형적인 구성은 다음과 같습니다:

필터기능
기울기전체 스펙트럼에 걸친 따뜻한/밝은 기울기 조정
저음초저음 및 저음 영역을 위한 로우 셀프 부스트 또는 감소
고음고음 및 초고음을 위한 하이 셀프 부스트 또는 감소
이어게인이도 공명 영역(약 2–3 kHz) 부근의 피킹 필터

슬라이더를 드래그하면 타겟 곡선 형태가 즉시 바뀝니다. 변경 사항은 원본 타겟 위에 적용되며, 대체하지는 않습니다. 기준선 버튼으로, 적용한 조정 효과만 따로 확인합니다. (곡선 다루기 참고)

프리셋​

대부분의 배포 환경은 슬라이더 조합을 한 번에 적용하는 원클릭 프리셋(일반적으로 하만 2013, 2015, 2018)을 제공하여, 시작점으로 쓰기 좋습니다.

  • 프리셋을 클릭하면 해당 슬라이더 값을 불러옵니다.
  • 원한다면 거기서부터 추가로 조절할 수 있습니다.
  • 다른 프리셋을 클릭해 덮어쓰거나, 초기화하여 처음부터 다시 시작합니다.
파일이 아닌 뷰만 조정합니다

사용자의 커스터마이징은 브라우저 세션에 저장되며 공유 URL에 인코딩됩니다. 디스크에 있는 사이트의 원본 타겟 파일은 수정되지 않습니다. (공유 URL 없이) 페이지를 새로고침하면 기본 타겟으로 돌아갑니다.

어떤 타겟을 커스터마이즈할 수 있나요?​

사이트 운영자가 변경 가능하다고 지정한 타겟만 가능합니다. 사용자 선택이 아니라 설정 단계에서 정해지는 사항으로, 운영자는 특정 레퍼런스 타겟이 항상 수정되지 않은 상태로 표시되도록 제한할 수 있습니다. 슬라이더 아이콘이 표시되지 않는 타겟이라면 해당 배포 환경에서는 커스터마이징할 수 없습니다.

모든 설정 옵션을 포함한 전체 기능 문서는 타겟 커스터마이저에서 확인합니다.

선호도 경계​

다수의 청취 평가 연구에 따르면, 모두를 동시에 만족시키는 궁극의 타겟 커브는 존재하지 않으며, 그 대신 대부분의 청취자가 만족스럽다고 느끼는 튜닝의 범위가 존재합니다. modernGraphTool은 이 범위를 그래프 위에 음영 처리된 영역인 선호도 경계로 그립니다.

그래프 툴바에서 선호도 경계를 켜고 끌 수 있으며, 이를 활성화하면,

  • 수용 가능한 영역의 상한과 하한을 나타내는 회색 영역이 나타납니다.
  • 이 영역은 설정된 기준 타겟(주로 Diffuse Field 타겟)에 고정되어 있어, 다른 타겟을 추가하더라도 선호도 경계에는 영향이 없습니다.
  • 음영 영역 _안_에 있는 곡선은 연구 기반의 선호도 범위에 속하며, 밖으로 벗어나는 곡선은 대다수의 청취자가 선호하는 것보다 더 밝거나 어둡다는 뜻입니다.

이는 합격/불합격 판정을 내리기 위한 절대 기준이 아니며, 선호도 경계를 벗어나는 헤드폰도 여전히 듣기 좋을 수 있습니다. 안전하면서도 중립적인 사운드라기보다는, 좀 더 개성 있는 튜닝이라는 의미로 해석하는 것이 바람직합니다. "옳고 그름"을 따지기 보다는, "보편성에서 얼마나 벗어났는가"의 지표로 활용하는 것이 좋습니다.

선호도 경계는 운영자가 제공해야 하는 데이터 파일이 필요하므로, 이를 설정하지 않은 사이트에서는 선호도 경계 버튼이 나타나지 않습니다. 전체 기능 문서는 선호도 경계를 참조하십시오.

권장 워크플로우​

새로운 헤드폰을 사용해 볼 때의 전형적인 과정은 다음과 같습니다:

  1. 기기 패널에서 헤드폰을 불러옵니다.
  2. 레퍼런스로 사용할 좋아하는 타겟을 선택합니다(하만, 리뷰어 타겟 등).
  3. 두 곡선이 중역에서 정렬되도록 평균에서 정규화(기본값)합니다.
  4. 헤드폰의 전반적인 소리 특성이 적정 범위 안에 있는지 확인하려면 선호도 경계를 사용할 수 있습니다.
  5. 헤드폰의 차이를 일직선으로 정렬하여 _직접 비교_해보려면 타겟을 기준선으로 설정합니다.

그런 다음 이 차이가 환영할 만한 변화인지, 아니면 이퀄라이저로 보정하고 싶은 변화인지 결정합니다.
