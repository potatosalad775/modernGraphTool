# 왜 modernGraphTool인가?

오디오 측정 커뮤니티의 요구는 기존에 훌륭히 역할을 해온 도구들의 수준을 이미 뛰어넘었습니다.

modernGraphTool은 최신 웹 기술 위에 모듈화된 아키텍처로 처음부터 다시 작성된 도구로, 측정 데이터베이스 운영자와 오디오 애호가들이 커뮤니티와 함께 발전할 수 있는 도구를 갖도록 설계되었습니다.

## CrinGraph의 유산

[CrinGraph](https://github.com/mlochbaum/CrinGraph)는 2019년경 등장한 뒤 오디오 애호가 커뮤니티에서 주파수 응답 시각화의 사실상 표준으로 자리잡았습니다.

본래의 역할은 훌륭히 해냈습니다: 측정값을 불러오고, 커브를 겹쳐서 비교하고, 헤드폰을 비교하는 단순하면서도 효과적인 도구였죠.

squig.link 네트워크의 대부분의 측정 데이터베이스는 지금도 CrinGraph 또는 그 파생 버전을 사용하고 있으며, FR 데이터를 시각화하는 방식에 끼친 영향은 부정할 수 없습니다.

:::info[완전한 데이터 호환]
modernGraphTool은 CrinGraph를 '부정'하는 도구가 아니라 **계승**하는 도구입니다.

동일한 데이터 형식, 동일한 폴더 구조, 동일한 `phone_book.json`을 사용합니다. 기존 측정 데이터는 어떤 변환 없이도 그대로 동작합니다.
:::

## 어디서부터 어려워졌는가

커뮤니티가 성장하면서 도구에 대한 요구도 함께 커졌지만, CrinGraph의 본래 설계는 그런 요구들을 수용할 수 있도록 만들어지지 않았습니다.

### 단일 스크립트가 모든 것을 처리

CrinGraph의 핵심 기능 — 데이터 로딩, 그래프 렌더링, UI 구성, 상태 관리 — 은 수천 줄에 달하는 단일 JavaScript 파일 안에 모두 들어 있습니다. 관심사의 분리가 전혀 없습니다.

숙련된 개발자라도 다른 부분을 망가뜨리지 않고 특정 부분만 수정하는 것이 쉽지 않고, 프로그래밍 경험이 없는 운영자에게는 사실상 불가능에 가깝습니다.

### 스타일이 여기저기 흩어져 있음

시각적 외관은 변수 없는 여러 CSS 파일에 분산되어 있으며, JavaScript 전반의 수많은 `createElement` 호출 안 인라인 스타일과 결합되어 있습니다. 도구 전체를 테마로 입히기 위해 수정할 단일 파일이라는 것이 없습니다.

색상 스킴을 바꾸거나 외관을 자기 취향에 맞추는 것은 CSS와 JS 양쪽을 뒤져가며 모든 케이스를 놓치지 않기를 바라야 하는 작업이 됩니다.

### 포크는 갈라지고 기능은 파편화됨

애호가들이 새로운 기능 — 예를 들어 Diffuse Field 타겟에 대한 선호도 조정 — 을 원하게 되었을 때, 전체 프로젝트를 포크해 핵심 스크립트를 직접 수정해야 했습니다.

그렇게 각 포크는 자기만의 섬이 되었습니다. 한 포크에서 추가된 기능이 다른 포크로 전파되지 않았고, 운영자들은 어떤 기능 조합을 사용할지 골라야 했습니다 — 서로 조합할 방법도 없이 말이죠.

## modernGraphTool은 어떻게 다른가

modernGraphTool은 SvelteKit, TypeScript, D3.js로 완전히 새로 만들어진 그래프 도구입니다.

낡아가는 코드베이스에 패치를 쌓아올리는 대신, 커뮤니티가 필요로 하는 기능을 위해 처음부터 설계되었습니다.

| | CrinGraph / 포크 | modernGraphTool |
|---|---|---|
| **아키텍처** | 단일 JS 스크립트 (수천 줄) | 모듈화된 SvelteKit 컴포넌트 (TypeScript) |
| **테마** | 여러 CSS 파일 + 인라인 스타일 | CSS 변수 기반 단일 `theme.css` + [테마 생성기](/theme-generator) |
| **타겟 조정** | 포크에 따라 상이; 타겟 하나에 하드코딩된 3~4개 필터 | [타겟별 조정, 설정 가능한 필터, 필터 프리셋](./features/target-customizer) |
| **파라메트릭 EQ** | 포크마다 파편화된 외부 플러그인 스크립트 | [AutoEQ, 실시간 오디오 미리 듣기, 인터랙티브 EQ가 기본 제공](./features/equalizer) |
| **디바이스 PEQ 브리지** | [jeromeof의 외부 플러그인](https://github.com/jeromeof/devicePEQ) | [임포트 후 리팩터링된 기본 제공 컴포넌트](./features/device-peq) |
| **다국어** | 미지원 | 영어 + 한국어 ([컴파일 타임 i18n](https://inlang.com/m/gerre34r/library-inlang-paraglideJs), 확장 가능) |
| **성능** | 번들링되지 않은 vanilla JS | 트리 셰이킹과 코드 스플리팅으로 번들링 |
| **업데이트** | 수동 파일 교체 | CDN 모드: 재업로드 없이 자동 업데이트 |
| **데이터 형식** | `phone_book.json` + FR 텍스트 파일 | 동일한 형식 — 그대로 호환 |

## 운영자를 위한 설계

modernGraphTool은 측정 데이터베이스 운영자가 소스 코드를 건드리지 않고도 사이트를 설치, 커스터마이즈, 유지보수할 수 있도록 설계되었습니다.

- **하나의 파일로 설정** — [`config.js`](./guide-for-admins/customize-page)가 모든 것을 제어합니다: 초기 기기, 타겟, 정규화, 워터마크, 기능 토글 등.
- **하나의 파일로 테마** — [`theme.css`](./guide-for-admins/customize-page)는 그래프 색상, UI 액센트 등을 위한 CSS 커스텀 속성을 사용합니다. 온라인 [테마 생성기](/theme-generator)가 이를 자동으로 만들어 줍니다.
- **CDN 배포** — 페이지를 CDN에 연결하면 업데이트가 자동으로 전달됩니다. 데이터와 설정은 사용자의 서버에 그대로 유지됩니다.
- **데이터 호환성** — 기존의 `data/` 폴더와 `phone_book.json`을 그대로 사용합니다. 변환도, 마이그레이션 스크립트도 필요 없습니다.
- **이중 호스팅** — 원한다면 기존 CrinGraph와 modernGraphTool을 같은 도메인에서 함께 운영할 수 있어 [점진적으로 전환](./database-tips/dual-hosting/main-cringraph)할 수 있습니다.

## 활발한 유지보수

CrinGraph와 그 주요 포크들은 현재 활발히 유지보수되고 있지 않습니다. modernGraphTool은 다릅니다 — 개발자 본인이 여러분과 같은 'Squiggler'이자 오디오 리뷰어입니다.

새로운 기능, 버그 수정, 개선 사항이 꾸준히 릴리스됩니다. CDN 모드를 사용한다면 어떤 수동 작업도 없이 자동으로 전달됩니다.

:::info[시작할 준비가 되셨나요?]
환경을 구성하려면 [환경 구성 가이드](./guide-for-admins/setup-env)로, 이미 modernGraphTool을 실행 중이라면 [페이지 커스터마이징](./guide-for-admins/customize-page)으로 바로 이동하세요.
:::
