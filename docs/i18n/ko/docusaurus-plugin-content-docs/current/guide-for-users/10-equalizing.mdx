# 오디오 이퀄라이징

**이퀄라이저 패널**에서 파라메트릭 EQ를 디자인하고, 실제 오디오로 미리 들어보고, 호환되는 하드웨어 기기로 전송할 수 있습니다. 이 문서는 사용자 가이드 수준의 소개용 텍스트입니다. 전체 기능 참조는 [Equalizer](../features/equalizer)에서 확인할 수 있고, 하드웨어 연동은 [Device PEQ](../features/device-peq)에서 다룹니다.

## 한 문단으로 설명하는 EQ의 용도

헤드폰 측정 데이터는 사용자의 취향에 비해 과하거나 미미한 대역을 정확히 보여줍니다. **EQ는 이러한 차이를 오디오 신호가 헤드폰에 도달하기 전에 디지털 방식으로 상쇄**합니다. 5kHz의 피크를 낮추거나, 서브베이스를 약간 올리거나, 보컬의 치찰음을 부드럽게 완화합니다. 구형 스테레오 앰프의 베이스/트레블 노브와 원리는 같지만, 훨씬 더 정밀하고 세밀하게 작동합니다.

modernGraphTool의 EQ 패널은 임의의 오디오 파일이 아닌 측정값을 기반으로 작동합니다. 불러온 기기를 선택하고 필터를 추가하면 그래프 위에 수정된 측정 곡선이 나타납니다. 또한 재생 기능을 켜면 브라우저 안의 테스트 오디오 샘플에 필터가 적용된 소리를 직접 들어볼 수 있습니다.

## 패널 살펴보기

이퀄라이저 패널(세 번째 메뉴 아이콘을 클릭하거나 데스크톱에서 **3** 누름)을 열면 다음 항목이 표시됩니다.

- 상단의 **마스터 활성화 스위치** — 설정한 필터를 유지한 채 전체 EQ를 일시적으로 끄고 싶을 때 사용합니다.
- 스위치 옆의 **폰/타겟 선택기** — EQ로 수정할 대상 곡선을 지정합니다.
- **필터 목록** — 실제 EQ 대역들입니다.
- 하단의 접을 수 있는 세 가지 섹션: **AutoEQ**, **Audio player**, **Device PEQ**.

## 직접 EQ 만들기

필터 목록을 클릭하여 필터를 추가합니다. 각 필터는 다음 항목을 갖습니다.

- **유형** — Peaking (특정 주파수를 중심으로 솟아오르거나 파이는 형태), Low Shelf (지정 주파수 아래의 모든 대역), High Shelf (지정 주파수 위의 모든 대역).
- **주파수** — 필터의 중심이 되는 X축 위치입니다.
- **게인** — 양수는 증폭, 음수는 감소를 의미합니다(dB 단위).
- **Q** — 효과가 적용되는 대역폭을 결정합니다. Q값이 클수록 폭이 좁아집니다.

값을 조정하면 그래프가 실시간으로 갱신됩니다. 기기와 EQ가 결합된 새로운 곡선이 나타나므로, 보정된 곡선과 타겟을 실시간으로 비교할 수 있습니다. 다수의 대역을 증폭할 때 생기는 디지털 클리핑을 막으려고 프리앰프 게인이 자동으로 계산됩니다.

최소한의 작업 흐름은 다음과 같습니다.

1. 폰/타겟 선택기에서 불러온 폰을 선택합니다.
2. 마스터 활성화 스위치를 켭니다.
3. 필터를 추가하여 Peaking으로 설정한 후, 수정할 주파수로 드래그하고 게인을 조절합니다.
4. 그래프를 확인하여 보정된 곡선이 타겟과 일치하거나 근접하는지 확인합니다.

## AutoEQ

필터를 직접 디자인하고 싶지 않다면, **AutoEQ** 섹션을 엽니다. 다음 세 단계로 진행됩니다.

1. 일치시킬 **타겟을 선택**합니다(미리 불러온 타겟만 가능).
2. **AutoEQ 버튼을 클릭**하면 기기와 타겟 간의 차이를 분석해 이를 최소화하는 필터 세트를 자동으로 생성합니다.
3. 필터 목록에 필터를 **적용**합니다. 원한다면 추가로 세밀하게 조정할 수 있습니다.

AutoEQ는 결과물에 Peaking 외에 Low Shelf나 High Shelf 필터도 포함하며, 이를 활용하면 보통 더 적은 필터 대역으로 더욱 평탄한 일치 결과를 얻습니다.

## 오디오 미리 듣기

**Audio player** 섹션을 열면 테스트 오디오 샘플에 적용된 EQ 소리를 들어볼 수 있습니다. 재생/일시정지와 음소거 토글로 보정 전후의 소리를 빠르게 비교합니다.

:::warning[브라우저 자동 재생]
대부분의 브라우저는 사용자가 페이지와 상호 작용할 때까지 오디오 자동 재생을 차단합니다. 재생 버튼을 눌러도 반응이 없다면, 페이지 아무 곳이나 먼저 클릭한 후 다시 시도하십시오. 이는 브라우저 정책이며 툴의 버그가 아닙니다.
:::

플레이어는 Web Audio API를 사용하므로, 실행 중일 때(특히 다수의 EQ 대역이 활성화된 경우) CPU 사용량이 약간 늘어납니다.

## 기기 PEQ — 하드웨어로 EQ 전송

일부 USB DAC, 블루투스 어댑터, 그리고 소수의 헤드폰에는 소프트웨어가 아닌 하드웨어 안에서 직접 EQ를 처리하는 **내장 파라메트릭 EQ 슬롯**이 있습니다. **Device PEQ** 섹션에서 modernGraphTool이 WebUSB, WebSerial, WebHID, WebBluetooth로 이러한 기기들과 직접 통신하여 기기 자체의 EQ 메모리에 필터 목록을 저장합니다. 이렇게 전송된 EQ는 컴퓨터 연결을 끊어도 기기에 그대로 남습니다.

지원 기기 목록, 연결 방법, 문제 해결에 관한 전체 정보는 [Device 파트](../features/device-peq) 문서를 참조하십시오. 간단히 요약하면 다음과 같습니다.

- **Chrome 및 Edge**는 이 기능이 요구하는 웹 API를 가장 완벽하게 지원합니다.
- **Firefox**는 일부 기기 유형에서만 작동합니다.
- **Safari**는 WebUSB를 전혀 지원하지 않으므로 Device PEQ 기능이 작동하지 않습니다.

## 가져오기 및 내보내기

현재 구성한 필터 세트를 텍스트 파일(일반적인 EQ 형식과 호환)로 **내보내기**하거나 나중에 다시 **가져오기** 합니다. 다음 상황에서 유용합니다.

- 설정한 EQ를 다른 사람과 공유할 때.
- 여러 헤드폰의 프리셋 라이브러리를 보관할 때.
- modernGraphTool과 시스템 전역 EQ 애플리케이션 사이에서 EQ 설정을 옮길 때.

가져오기/내보내기 기능은 필터 목록 안에 있습니다. 지원하는 형식의 세부 정보는 [Equalizer](../features/equalizer)를 참조하십시오.
