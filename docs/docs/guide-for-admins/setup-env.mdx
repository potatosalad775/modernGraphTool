# Setting Up Your Environment

This guide explains the basic environment setup for using and managing modernGraphTool.

## Prerequisites

1.  **modernGraphTool Project Files**: The built application files containing the core functionalities for visualizing and analyzing frequency response data.
2.  **Text Editor**: A program for modifying various configuration files (`.js`, `.css`, `.json`). [Visual Studio Code (VS Code)](https://code.visualstudio.com/) is recommended.

## Option A: Using the Pre-built Release (Recommended)

This is the simplest option. No programming knowledge is required.

1.  Download the [latest release zip file](https://github.com/potatosalad775/modernGraphTool/releases) from the [modernGraphTool GitHub Release Page](https://github.com/potatosalad775/modernGraphTool/releases).
2.  Extract the downloaded zip file. The extracted folder contains everything you need to deploy.
3.  Customize the configuration files to suit your needs:
    - `config.js` — Application settings (see [Customizing the Page](./customize-page))
    - `theme.css` — Graph color theme (see [Customizing Page Styles](./customize-page#customizing-page-styles-themecss))
    - `data/` — Your measurement data and target curves (see [Managing Data](./manage-data))
4.  Upload the entire folder to your web server via FTP or any file transfer method.

## Option B: Building from Source (For Developers)

If you want to modify the source code or build from the latest development version:

1.  Ensure you have [Node.js](https://nodejs.org/) (v18 or later) installed on your system.
2.  Clone the repository:
    ```bash
    git clone https://github.com/potatosalad775/modernGraphTool.git
    cd modernGraphTool
    ```
3.  Install dependencies:
    ```bash
    npm install
    ```
4.  Customize the configuration:
    - `static/config.js` — Application settings
    - `static/theme.css` — Graph color theme
    - `static/data/` — Your measurement data and target curves
5.  Build the project:
    ```bash
    npm run build
    ```
6.  The built output will be in the `dist/` folder. Deploy this folder to your web server.

:::tip[Local Preview]
You can preview your changes locally before deploying:
- **From source**: Run `npm run dev` to start a development server at `http://localhost:5173`
- **Built output**: Run `npm run preview` to serve the `dist/` folder locally
:::

## Installing a Text Editor

While various text editors are available, using [Visual Studio Code (VS Code)](https://code.visualstudio.com/) is recommended for its convenient features like code highlighting and auto-completion.

VS Code can be downloaded for free by anyone and supports Windows, macOS, and Linux.

1.  Go to the [VS Code website](https://code.visualstudio.com/), download the installer for your operating system, and install it.
2.  Open the modernGraphTool project folder (or the extracted release folder) in VS Code.
3.  You can now edit configuration files and manage your data.

## Project Structure

After extracting the release (or building from source), the key files and folders are:

```
dist/                     # (or extracted release folder)
├── config.js             # Application settings
├── theme.css             # Graph color theme
├── index.html            # HTML entry point (editable for metadata)
├── data/
│   ├── phones/           # Measurement data files (.txt)
│   ├── target/           # Target curve data files (.txt)
│   └── phone_book.json   # Device list definition
└── assets/               # Static assets (images, strings, etc.)
```

:::info[For Source Builders]
When building from source, editable files are in the `static/` folder. The HTML entry point in the source is `src/app.html` (SvelteKit template). After building, these are compiled and output to `dist/`.
:::
