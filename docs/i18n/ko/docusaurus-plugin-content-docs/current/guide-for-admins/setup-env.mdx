# 환경 구성하기

이 페이지는 설치 과정에서의 선택지를 정리하고 올바른 배포 옵션으로 안내합니다. 각 옵션에는 아래 표에서 링크된 전용 가이드가 있습니다.

## 준비물

1.  **modernGraphTool 프로젝트 파일** — 선택한 배포 옵션에 따라 [사전 빌드 릴리스](https://github.com/potatosalad775/modernGraphTool/releases) 또는 소스 저장소의 클론이 필요합니다.
2.  **텍스트 에디터** — `config.js`, `theme.css`, 기타 설정 파일 수정을 위한 에디터. 구문 강조와 자동 완성을 제공하는 [Visual Studio Code](https://code.visualstudio.com/) 사용을 권장합니다.

## 배포 옵션

modernGraphTool은 세 가지 배포 경로를 지원합니다. 업데이트와 커스터마이징을 얼마나 직접 제어하고 싶은지에 따라 선택하세요:

| 옵션 | 적합 대상 | 업데이트 | 커스터마이징 | 하위 디렉토리 호스팅 |
|---|---|---|---|---|
| **[CDN 배포](./deployment/cdn)** (권장) | 대부분의 운영자 | 자동 | 설정, 테마, 데이터 | 설정 한 줄 필요 (`BASE_PATH`) |
| **[사전 빌드 릴리스](./deployment/prebuilt)** | 통제된 배포, 에어갭 환경 | 수동 (재다운로드) | 설정, 테마, 데이터, HTML 메타데이터 | 즉시 동작 |
| **[소스에서 빌드](./deployment/from-source)** | 개발자, 포크 | 수동 (git pull + 빌드) | 전부 | 즉시 동작 |

### [CDN 배포](./deployment/cdn)

서버에는 최소한의 `index.html` 로더와 `config.js`, `theme.css`, `data/`, `assets/`만 호스팅됩니다. 애플리케이션 코드 자체는 jsDelivr에서 제공되며 새 버전이 나오면 자동으로 업데이트됩니다. 가장 간단한 경로입니다 — 수동 업데이트도, 재업로드도 없습니다.

사이트가 `/headphones/`와 같은 하위 디렉토리에 있다면 (squig.link에서 흔한 패턴 — 같은 서브도메인에 루트 이어폰 데이터베이스와 `/headphones/` 헤드폰 데이터베이스를 함께 운영), `config.js`에 `CDN_MODE.BASE_PATH`를 설정해야 합니다 — 설정 한 줄이면 되지만 놓치기 쉽습니다. 자세한 내용은 [CDN 배포 가이드](./deployment/cdn#why-base_path-is-almost-always-required)를 참고하세요.

### [사전 빌드 릴리스](./deployment/prebuilt)

완전한 `dist/` 폴더를 다운로드해 서버에 업로드합니다. 어떤 버전이 언제 동작할지 정확히 제어할 수 있습니다 — jsDelivr에 런타임 의존성이 없고, 에어갭 환경에서도 동작하며, 하위 디렉토리 호스팅도 추가 설정 없이 바로 됩니다. 단점은 새 릴리스로 업데이트하려면 다시 내려받아 다시 업로드해야 한다는 점입니다.

### [소스에서 빌드](./deployment/from-source)

저장소를 클론하고, 원하는 대로 커스터마이즈한 뒤, 자신만의 `dist/` 폴더를 빌드합니다. 이것은 개발자 경로입니다 — `config.js`에서 노출하는 범위를 넘는 변경이 필요한 경우 사용하세요.

## 텍스트 에디터 설치

다양한 텍스트 에디터가 있지만, 코드 하이라이팅과 자동 완성 같은 편리한 기능을 제공하는 [Visual Studio Code](https://code.visualstudio.com/) 사용을 권장합니다.

VS Code는 누구나 무료로 다운로드해 사용할 수 있으며, Windows, macOS, Linux를 모두 지원합니다.

1.  [VS Code 웹사이트](https://code.visualstudio.com/)에 접속하여 운영체제에 맞는 설치 파일을 다운로드하고 설치합니다.
2.  VS Code에서 modernGraphTool 프로젝트 폴더 (또는 압축 해제한 릴리스 폴더)를 엽니다.
3.  이제 설정 파일을 수정하고 데이터를 관리할 수 있습니다.
